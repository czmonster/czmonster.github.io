实现对gzip压缩过的request进行解压缩

2016-01-11


好久没有写博客了，把之前在公司Wiki上写的一篇文章放下来。
场景：由于特殊需求，Client对request进行了gzip压缩。而原有的apache服务器在中间做了一次解压缩后传给了后台server（请求数据体积大，后台不支持处理压缩过的请求）。而最近准备用Nginx替换掉笨重的apache。查资料发现Nginx和压缩相关的有gzip和gunzip。不过在这里都用不上。

  • gzip实现的功能有：1、对request进行压缩。2、对response进行压缩。
  • gunzip实现的功能有：对response进行解压缩。(gunzip模块默认编译时并不安装)
    一番google之后发现nginx确实没有自带这样的功能…有介绍使用lua大法去解决这个问题…nginx lua模块安装参考lua-nginx-module
    用lua进行gzip压缩需要借助lua-zlib （https://github.com/brimworks/lua-zlib）C语言编写。
    `cd /usr/local/src yum install git cmake git clone https://github.com/brimworks/lua-zlib.git cd lua-zlib/ cmake -DLUA_INCLUDE_DIR=/usr/local/include/luajit-2.0 -DLUA_LIBRARIES=/usr/local/lib -DUSE_LUAJIT=ON -DUSE_LUA=OFF make && make install luajit 然后在实时编辑器里输入 require 'zlib' 回车，如果没有报错则表示该库已成功安装。
    `

以下是lua脚本inflate.lua：
local zlib = require “zlib”
file = io.open(“/etc/nginx/pos_data.txt”,”a”)
local encoding = ngx.req.get_headers()[“Content-Encoding”]
if encoding == “gzip” then
—这行一定不能漏，要不然下面获得的是个nil
ngx.req.read_body()
local body =ngx.req.get_body_data()
if body then
local stream = zlib.inflate()
uncompress = stream(body)
—file:write(uncompress)
ngx.req.set_body_data(uncompress)
ngx.req.clear_header(“Content-Encoding”)
end
end
file:close()
由于找不到合适的断点方式只好把一些过程数据写入到文件中了.注意：记得把文件权限打开..要不然写文件时会报错..
2016/01/06 12:40:10 [error] 25988#0: *4 lua entry thread aborted: runtime error: /etc/nginx/inflate.lua:17: attempt to index global ‘file’ (a nil value)
上面写好了lua脚本，那么要在ngxin中使用。修改nginx.conf
ocation /XXX/ {
proxy_pass http://ip:port;
rewrite_by_lua_file /etc/nginx/inflate.lua;
}
测试异常，通过查看nginx的error.log是因为找不到zlib，怀疑是路径问题。需要把lua-zlib-master/zlib.so放到指定目录（我放在和inflate.la同目录下）然后测试通过。
