唯一约束和唯一索引

2015-09-27


记录下之前生产发布出的一个事故，反思。
原因是因为业务变动，一张表的唯一约束需要干掉。发布前执行了DROP CONSTRAINT语句，之后数据库压力瞬间上来。因为这张表的是张大表，操作又频繁。而把删除唯一约束的同时把一个很关键的索引给干掉了。造成这起事故的最根本原因是对唯一约束和索引的认识不够。经过一系列的测试后，得出以下两个学习结果：

  • oracle在创建唯一约束的时会创建唯一索引，如果之前有索引则会使用之前存在的索引。
  • 在删除唯一约束时，oracle会级联删除oracle自动生成的唯一索引。如果这个索引是手动先创建的，则不会被删除
    附上一些测试脚本：
    —建测试表
    create table GE_TEST
    (
    id                   NUMBER(19) not null,
bill_code            VARCHAR2(255 CHAR)
    );
    —创建唯一约束
    alter table ge_test add constraint idx_bc unique(bill_code);
    —删除约束
    alter table ge_test drop constraint idx_bc
    —创建索引
    create index idx_bc on ge_test(bill_code);
    —删除索引
    drop index idx_bc
    —查看约束
    SELECT constraint_name,constraint_type,table_name,index_owner,index_name FROM user_constraints where table_name = ‘GE_TEST’;
    —查看索引
    SELECT index_name,index_type FROM user_indexes t WHERE t.table_name=’GE_TEST’;

删除唯一约束保留索引正确姿势：
Alter table table_name Drop Constraint idx_name keep index

为了避免再发生这种情况。写了个Ruby脚本去检查每次要执行的DDL脚本。把删除索引和删除唯一约束都定义为危险动作。

#!/usr/bin/env ruby

filename = ARGV[0]
enc = ARGV.length > 1 ? ARGV[1].to_s : 'utf-8'
if File.exist?(filename)
  file = File.open(filename, "r:#{enc}")
  file.each_line { |line| print "#{file.lineno} : ", line if /DROP +(CONSTRAINT|INDEX)/i =~ line }
  file.close
  puts "Done"
else
  puts "No file: #{filename}"
end
